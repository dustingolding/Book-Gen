# DailyCast v4 Implementation Pack

This document maps the v4 DailyCast concept to the current repository implementation.

## Stage Order

1. `factpack_flow_v2`
2. `rag_index_flow`
3. `script_gen_flow`
4. `realism_flow_v2`
5. `verify_flow_v2`
6. `elevenlabs_flow` (gated by `ELEVENLABS_ENABLED`)
7. `publish_flow_v2`

Orchestrator: `app/flows/dailycast_flow_v4.py` (`slw-dailycast-parent-v4`).

## Core Contracts

- Show identity contract: `config/show_identity.yaml`
- Coverage weighting + seasonality: `config/coverage_weights.yaml`, `config/seasonality.yaml`
- FactPack schema: `schemas/factpack.schema.json`
- Script schema: `schemas/script.schema.json`
- QA report schema: `schemas/qa_report.schema.json`

## Storage and Lineage

- Object store source of truth: MinIO bucket (`S3_BUCKET`, default `slw-dailycast`)
- LakeFS branch per run: generated by `app/services/lakefs_refs.py`
- Stage artifacts are committed after each flow stage in v4 parent flow.

## NotebookLM Artifact Set

The FactPack stage writes these docs to `notebooklm/<run_date>/`:

- `00_show_contract.md`
- `00_prompt_contract.md`
- `01_games.md`
- `02_player_stats.md`
- `03_team_trends.md`
- `05_major_news.md`
- `06_today_matchups.md`
- `07_season_outlooks.md`
- `08_editorial_notes.md`

## Runtime Entry Points

- Default v4 run: `python -m app.cli run-flow --run-date YYYY-MM-DD`
- Explicit v4 run: `python -m app.cli run-flow-v4 --run-date YYYY-MM-DD`
- Legacy run (fallback): `python -m app.cli run-flow-legacy --run-date YYYY-MM-DD`
